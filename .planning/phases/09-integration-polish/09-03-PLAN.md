# Plan 09-03: Edge Case Handling and Retina Normalization

## Objective
Handle retina (2x/3x) screenshots and edge cases for production-ready output.

## Current State
- detectScaleFactor() in main.ts uses simple threshold (>2000px = 0.5x)
- No 3x retina detection
- No device-specific size detection

## Edge Cases to Handle

### 1. Retina Display Detection
Common retina dimensions:
- iPhone 15 Pro: 2556x1179 (3x) → normalize to 852x393
- MacBook Pro 14": 3024x1964 (2x) → normalize to 1512x982
- iPad Pro 12.9": 2732x2048 (2x) → normalize to 1366x1024

Detection strategy:
- Check common device resolutions
- Use aspect ratio + dimension analysis
- Fall back to dimension-based heuristic (>2500 = 3x, >1800 = 2x)

### 2. Empty Analysis Response
- If Claude returns no elements, show helpful message
- Suggest using higher contrast screenshot

### 3. Invalid Element Data
- Skip elements with missing bounds
- Log warnings for debugging
- Continue processing valid elements

### 4. Viewport Dimension Validation
- Cap maximum dimensions (prevent massive frames)
- Minimum dimensions (prevent invisible frames)
- Default fallback if viewport missing

## Implementation Steps

### Step 1: Improve Retina Detection
Update detectScaleFactor in main.ts:
- Check against known device resolutions
- Support 1x, 2x, 3x scale factors
- Add manual override option for edge cases

### Step 2: Add Dimension Validation
Add validateViewport utility:
- Clamp dimensions to reasonable range (100-4000px)
- Apply scale factor normalization
- Return validated viewport

### Step 3: Robust Element Filtering
Update FigmaGenerator.generate:
- Pre-filter elements with invalid bounds
- Log filtered elements for debugging
- Continue with valid elements

### Step 4: User Feedback for Edge Cases
Update UI to show:
- "Retina screenshot detected (2x)" message
- Warning for low element count
- Helpful suggestions for better results

## Success Criteria
1. 2x retina screenshots produce correctly sized output
2. 3x retina screenshots produce correctly sized output  
3. Invalid elements skipped gracefully
4. User sees helpful messages for edge cases
5. Generated designs immediately usable (no size cleanup)
