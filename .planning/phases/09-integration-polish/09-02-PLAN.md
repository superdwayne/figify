# Plan 09-02: Performance Optimization and Batching

## Objective
Ensure complex screenshots with 20+ elements process without freezing the UI.

## Current State
- FigmaGenerator has yieldToEventLoop() for preventing freezes
- createBatches() utility exists but isn't fully utilized
- Progress reporting callback exists
- batchSize option (default 10) is defined

## Performance Concerns
1. Large element trees may cause UI freezes during processing
2. Font preloading happens all at once
3. No throttling on progress updates (could flood message channel)

## Implementation Steps

### Step 1: Throttle Progress Updates
Update FigmaGenerator.reportProgress:
- Throttle to max 10 updates per second
- Always report first and last updates
- Prevent UI message flooding

### Step 2: Batch Font Preloading
Update StyleApplier.preloadFonts:
- Group fonts into batches
- Yield between batches for large font sets

### Step 3: Optimize Tree Processing
Update processElementWithChildren:
- Add periodic yields during deep tree traversal
- Use existing batchSize option for grouping root elements

### Step 4: Add Generation Timeout
Implement timeout safety:
- Maximum generation time (30 seconds default)
- Graceful abort with partial results if timeout exceeded

## Success Criteria
1. 20+ element screenshots process without visible UI freeze
2. Progress updates smoothly (no flickering)
3. Generation completes within reasonable time (<10s typical)
4. Timeout prevents runaway generation

## Files to Modify
- `src/main/generator/index.ts` - Progress throttling, timeout
- `src/main/generator/styleApplier.ts` - Font batching
