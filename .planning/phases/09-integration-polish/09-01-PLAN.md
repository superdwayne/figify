# Plan 09-01: End-to-End Flow Integration

## Objective
Connect the UI analysis results to Figma generation, creating the complete workflow: paste screenshot → analyze → generate Figma design.

## Current State
- ImageCapture has Claude analysis working (useClaude hook)
- AnalysisResult displays detected elements
- FigmaGenerator creates Figma nodes from UIAnalysisResponse
- Message protocol has GENERATE_DESIGN, PROGRESS, GENERATION_COMPLETE types
- Main thread has handleGenerateDesign handler ready

## Missing Pieces
1. "Generate Design" button in AnalysisResult component
2. useGeneration hook to communicate with main thread
3. Progress display during generation
4. Success/error feedback after generation completes

## Implementation Steps

### Step 1: Create useGeneration Hook
Create `src/ui/hooks/useGeneration.ts`:
- State: isGenerating, progress, error, result
- generateDesign(analysis) function sends GENERATE_DESIGN to main thread
- Listen for PROGRESS and GENERATION_COMPLETE messages
- Return state and actions

### Step 2: Add Message Listeners to App.tsx
Extend handleMessage to process:
- PROGRESS messages → update progress state
- GENERATION_COMPLETE → handle success/error

### Step 3: Update AnalysisResult with Generate Button
Add "Generate in Figma" button:
- Disabled while generating
- Shows progress indicator during generation
- Success message with element count
- Error display if generation fails

### Step 4: Wire Up ImageCapture
Pass generation state from useGeneration to AnalysisResult:
- Progress callback for live updates
- Result display after completion

## Success Criteria
1. User can click "Generate in Figma" after analysis
2. Progress indicator shows during generation
3. Success message shows element count after completion
4. Errors display clearly if generation fails
5. Generated frame selected and zoomed to view

## Files to Create/Modify
- CREATE: `src/ui/hooks/useGeneration.ts`
