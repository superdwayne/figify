<task type="auto">
  <name>Task 2: Create Claude client service and error utilities</name>
  <files>src/ui/services/claude.ts</files>
  <action>
Create `src/ui/services/claude.ts` with three exports:

1. **createClaudeClient(apiKey: string)** - Factory function:
   - Returns new Anthropic client instance
   - MUST include `dangerouslyAllowBrowser: true` (required for browser CORS)
   - Takes user's API key from useApiKey hook

2. **getErrorMessage(error: unknown): string** - Error translator:
   - Check `error instanceof Anthropic.APIError` first
   - Map status codes to user-friendly messages:
     - 400: "Invalid request. Please try a different image."
     - 401: "Invalid API key. Please check your settings."
     - 403: "API key does not have permission for this operation."
     - 413: "Image is too large. Please use a smaller image (max 5MB)."
     - 429: "Rate limit exceeded. Please wait a moment and try again."
     - 500: "Claude is experiencing issues. Please try again later."
     - 529: "Claude is temporarily overloaded. Please try again in a few minutes."
     - default: "An unexpected error occurred. Please try again."
   - Check for AbortError: return empty string (no message for cancelled)
   - Check for network errors (message includes 'network' or 'fetch'): "Network error. Please check your internet connection."
   - Fallback: "An unexpected error occurred. Please try again."

3. **analyzeImage(client, imageBase64, mimeType, signal?)** - API call wrapper:
   - Takes Anthropic client instance (allows reuse)
   - Takes base64 image string (without data: prefix)
   - Takes mimeType ('image/png', 'image/jpeg', 'image/webp')
   - Takes optional AbortSignal for cancellation
   - Calls `client.messages.create()` with:
     - model: 'claude-sonnet-4-5-20250929'
     - max_tokens: 4096
     - messages with image content block (type: 'base64', media_type, data)
     - Simple prompt: "Analyze this UI screenshot and describe what you see."
   - Pass signal to options object: `{ signal }`
   - Extract text response from `message.content[0]`
   - Return string (the analysis text)

Note: The prompt is intentionally simple for Phase 4. Phase 5 will implement proper structured prompts for element detection.
  </action>

Run `pnpm typecheck` to verify TypeScript compiles without errors.

<task type="auto">
  <name>Task 3: Create base64 conversion utility</name>
  <files>src/ui/utils/base64.ts</files>
  <action>
Create `src/ui/utils/base64.ts` with one export:

**uint8ArrayToBase64(uint8Array: Uint8Array): string**

Implementation:
- Loop through Uint8Array building a binary string
- Use `String.fromCharCode(uint8Array[i])` for each byte
- Return `btoa(binary)` (standard base64 encoding)

This is needed because:
- Phase 3's useImageCapture provides `capturedImage.uint8Array`
- Claude API expects base64-encoded image data (not data URL, not Uint8Array)
- The conversion strips the data URL prefix that would cause API errors

Keep this utility simple - no chunking needed for typical screenshot sizes.
  </action>
